📋 The Customization Stack — How It All Works

Instructions, Skills, Agents & Hooks — explained with examples

📋
Instructions
Guide Copilot's behavior per project — always active
Always On ▸

What They Do

  • Shape Copilot's tone, style, coding conventions
  • Scoped per project, workspace, or user
  • Always included in every agent context automatically
  • Monorepo discovery — finds instructions in parent folders

Where They Live

📁 .github/
  📄 copilot-instructions.md ← project-level
📁 .vscode/
  📄 instructions.md ← workspace-level
📁 ~ (User home)
  📄 copilot-instructions.md ← user-level (global)
💡 Write instructions like you're onboarding a new developer — tech stack, conventions, what to avoid.
🛠️
Skills (Prompt Files)
Reusable prompts as /slash commands
/command ▸

What Are They?

  • Reusable prompt templates saved as .prompt.md
  • Appear as slash commands in chat: /review, /test-gen
  • Can include variables and file references
  • Shareable across the team via version control

File Structure

📁 .github/
  📁 prompts/
    📄 review.prompt.md
    📄 test-gen.prompt.md
    📄 refactor.prompt.md

Example: review.prompt.md

---
description: Review code for best practices
---
Review the selected code for:
- Performance issues
- Error handling gaps
- Naming conventions
Suggest improvements with code examples.

Custom Agents
Specialized AI personas for domain tasks
@invoke ▸

How It Works

📝 Define
.agent.md
→
🧠 Set instructions
& allowed tools
→
🪝 Attach hooks
(optional)
→
🚀 Invoke
@agent-name

Example: .agent.md

---
name: security-reviewer
description: Reviews code for OWASP Top 10
tools: ["codebase", "grep_search", "read_file"]
---
You are a security expert. Analyze for
SQL injection, XSS, CSRF vulnerabilities.

Capabilities

  • Custom tools — restrict which tools the agent can use
  • Scoped instructions — agent-specific prompt, doesn't leak to other chats
  • Hooks — attach pre/post scripts (see Hooks below)
  • Nested subagents — agents can invoke other agents
  • AI generation — describe in natural language → auto-generate
🪝
Hooks
Automated pre/post scripts around agent actions
Auto-run ▸

How Hooks Work

🪝 Pre-hook
runs before
→
🤖 Agent Action
tool call / edit
→
🪝 Post-hook
runs after

Hook Types

HookFires WhenUse Case
preCommandBefore terminal commandLint, validate
postCommandAfter terminal commandLog, notify, cleanup
preEditBefore file editBackup, format check
postEditAfter file editRun tests, type-check

Example in .agent.md

---
name: safe-deployer
hooks:
  preCommand:
    command: "npm run lint"
    description: "Lint before any terminal cmd"
  postEdit:
    command: "npm test"
    description: "Run tests after file edits"
---
🔑 Hooks only run when their agent is active. Enable via: chat.useCustomAgentHooks
